# How did you interpret the user’s needs and implement them into your software design? Why is it so important to consider the user’s needs when designing? 

I had considered the client’s need for expanding to different operating platforms and operating systems. For example, the user may use different systems, each requiring different input and output devices. An example of an output device is a screen which can be small like a phone, or large, like a personal computer (PC) monitor. Additionally, input devices like touch screens and mouse, must be taken into account for both user and client needs. If we do not consider what the user may use, we may experience issues such as visuals that are too small or too large on a screen. It is also important to recognize that different input devices require different graphical user interfaces (GUI), which are compatible with the current device being used within reasonable limits.
